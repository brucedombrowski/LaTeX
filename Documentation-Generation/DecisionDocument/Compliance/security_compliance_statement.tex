\dmsection{Certificate Handling}

The software implements secure certificate selection and handling:

\begin{itemize}
    \item \textbf{EKU Requirements:} Certificates filtered by Extended Key Usage (Email Protection, Document Signing)
    \item \textbf{Government Prioritization:} DOD, NASA, and FPKI certificates prioritized in selection
    \item \textbf{X.509 OID Filtering:} Source code uses OIDs to filter certificates by purpose (e.g., 1.3.6.1.5.5.7.3.4 for Email Protection)
    \item \textbf{Read-Only Access:} Certificate store accessed in read-only mode
\end{itemize}

% ----------------------------------------------------------------------------
\dmsection{Security Controls}

The following security controls are implemented in the software:

\begin{itemize}
    \item \textbf{No Private Key Logging:} Private key material is never written to logs or output
    \item \textbf{Secure PIN Entry:} PIN prompts handled via Windows secure dialog (no application access)
    \item \textbf{Certificate Store Protection:} Read-only certificate store access prevents modification
    \item \textbf{Sensitive File Exclusions:} \texttt{.gitignore} excludes \texttt{*.pfx}, \texttt{*.p12}, \texttt{*.key}, and other sensitive file types
\end{itemize}
